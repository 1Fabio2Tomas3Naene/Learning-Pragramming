Modelo HTML

Curso de Fundamentos

Introdução

Todos os documentos HTML têm a mesma estrutura básica ou boilerplate que precisa estar no lugar antes que qualquer coisa útil possa ser feita. Nesta lição, exploraremos as diferentes partes deste boilerplate e veremos como tudo se encaixa.

Visão geral da lição

Esta seção contém uma visão geral dos tópicos que você aprenderá nesta lição.

  • Como escrever o texto básico para um documento HTML.
  • Como abrir documentos HTML no seu navegador.

Criando um arquivo HTML

Para demonstrar um modelo HTML, primeiro precisamos de um arquivo HTML para trabalhar.

Crie uma nova pasta no seu computador e nomeie-a html-boilerplate. Dentro dessa pasta, crie um novo arquivo e nomeie-o index.html.

Você provavelmente já está familiarizado com muitos tipos diferentes de arquivos, por exemplo, doc, pdf e arquivos de imagem.

Para que o computador saiba que queremos criar um arquivo HTML, precisamos anexar a .htmlextensão ao nome do arquivo, como fizemos ao criar o index.htmlarquivo.

Vale a pena notar que nomeamos nosso arquivo HTML index. Devemos sempre nomear o arquivo HTML que conterá a homepage do nosso site index.html. Isso ocorre porque os servidores web, por padrão, procurarão por uma index.htmlpágina quando os usuários acessarem nossos sites – e não ter uma causará grandes problemas.

O DOCTYPE

Toda página HTML começa com uma declaração doctype. O propósito do doctype é dizer ao navegador qual versão do HTML ele deve usar para renderizar o documento. A versão mais recente do HTML é HTML5, e o doctype para essa versão é <!DOCTYPE html>.

Os doctypes para versões mais antigas do HTML eram um pouco mais complicados. Por exemplo, esta é a declaração do doctype para HTML4:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

No entanto, provavelmente nunca desejaremos usar uma versão mais antiga do HTML, então sempre usaremos <!DOCTYPE html>.

Abra o index.htmlarquivo criado anteriormente no seu editor de texto e adicione-o <!DOCTYPE html>na primeira linha.

Elemento HTML

Após declararmos o doctype, precisamos fornecer um <html>elemento. Isso é o que é conhecido como elemento raiz do documento, o que significa que todos os outros elementos no documento serão descendentes dele.

Isso se torna mais importante mais tarde, quando aprendemos sobre manipulação de HTML com JavaScript. Por enquanto, saiba apenas que o <html>elemento deve ser incluído em todo documento HTML.

De volta ao index.htmlarquivo, vamos adicionar o <html>elemento digitando suas tags de abertura e fechamento, assim:

<!DOCTYPE html>
<html lang="en">
</html>

Notou a palavra langaqui? Ela representa um atributo HTML que é associado à tag HTML dada, ou seja, <html>neste caso. Esses atributos fornecem informações adicionais sobre elementos HTML. (Mais sobre isso HTML attributesna próxima lição.)

O que é o atributo lang?

langespecifica o idioma do conteúdo do texto naquele elemento. Este atributo é usado principalmente para melhorar a acessibilidade da página da web. Ele permite que tecnologias assistivas, por exemplo leitores de tela, se adaptem de acordo com o idioma e invoquem a pronúncia correta.

Elemento principal

O <head>elemento é onde colocamos meta-informações importantes sobre nossas páginas da web, e coisas necessárias para que nossas páginas da web sejam renderizadas corretamente no navegador. Dentro do <head>, não devemos usar nenhum elemento que exiba conteúdo na página da web.

De volta ao nosso index.htmlarquivo, vamos adicionar um <head>elemento com um <meta>elemento e um título dentro dele. O <head>elemento vai dentro do <html>elemento e deve ser sempre o primeiro elemento sob a <html>tag de abertura:

<!DOCTYPE html>

<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
  </head>
</html>

Elemento meta

Devemos sempre ter a <meta>tag com a codificação do conjunto de caracteres da página da web no <head>elemento: <meta charset="utf-8">.

Definir a codificação é muito importante porque garante que a página da web exibirá símbolos e caracteres especiais de diferentes idiomas corretamente no navegador.

Elemento de título

Outro elemento que sempre devemos incluir no cabeçalho de um documento HTML é o <title>elemento:

<title>My First Webpage</title>

O <title>elemento é usado para dar às páginas da web um título legível por humanos, que é exibido na aba do navegador da nossa página da web. Por exemplo, se você olhar para o nome da aba atual do seu navegador, ele dirá “HTML Boilerplate | The Odin Project”; este é o <title>do .htmlarquivo atual.

Se não incluíssemos um <title>elemento, o título da página da web seria o nome do arquivo por padrão. No nosso caso, seria index.html, o que não é muito significativo para os usuários; isso tornaria muito difícil encontrar nossa página da web se o usuário tivesse muitas abas do navegador abertas.

Há muitos outros elementos que podem ir dentro do cabeçalho de um documento HTML. No entanto, por enquanto, é crucial saber apenas sobre os dois elementos que cobrimos aqui. Apresentaremos mais elementos que vão para o cabeçalho ao longo do restante do currículo.

Elemento do corpo

O elemento final necessário para completar o boilerplate HTML é o <body>elemento. É aqui que todo o conteúdo que será exibido aos usuários irá - o texto, imagens, listas, links e assim por diante.

Para completar o boilerplate, adicione um <body>elemento ao index.htmlarquivo. O <body>elemento também vai dentro do <html>elemento e está sempre abaixo do <head>elemento, assim:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
  </head>

  <body>
  </body>
</html>

Visualizando arquivos HTML no navegador

O boilerplate HTML no index.htmlarquivo está completo neste ponto, mas como você o visualiza no navegador? Há algumas opções diferentes:

Usar o Google Chrome

In order to avoid branching our lesson’s instructions to accommodate for all of the differences between browsers, we are going to be using Google Chrome as our primary browser for the remainder of this course. All references to the browser will pertain specifically to Google Chrome. We strongly suggest that you use Google Chrome for all of your testing going forward.

  1. You can drag and drop an HTML file from your text editor into the address bar of your browser.

  2. You can find the HTML file in your file system and then double click it. This will open up the file in the default browser your system uses.

  3. You can use the terminal to open the file in your browser:

    • Ubuntu: Navigate to the directory containing the file and type google-chrome index.html
    • macOS: Navigate to the directory containing the file and type open ./index.html
    • WSL: Navigate to the directory containing the file and type explorer.exe index.html. Note, this will open up the file in the default browser your system uses.

Using one of the methods above, open up the index.html file we have been working on. You’ll notice the screen is blank. This is because we don’t have anything in our body to display.

Back in the index.html file, let’s add a heading (more on these later) to the body, and save the file:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
  </head>

  <body>
    <h1>Hello World!</h1>
  </body>
</html>

Now, if you refresh the page in the browser, you should see the changes take effect, and the heading “Hello World!” will be displayed.

VSCode shortcut

VSCode has a built-in shortcut you can use for generating all the boilerplate in one go. Please note that this shortcut only works while editing a file with the .html extension or a text file with the HTML language already selected. To trigger the shortcut, delete everything in the index.html file and just enter ! on the first line. This will bring up a couple of options. Press the Enter key to choose the first one, and voila, you should have all the boilerplate populated for you.

You may notice that the boilerplate produced by this shortcut includes a line we have not yet mentioned:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This is not something we need to know about until we discuss responsive design, an advanced topic involving different screen sizes which we will cover much later in the curriculum. For now, you can leave that line as it is.

It’s still good to know how to write the boilerplate yourself in case you find yourself using a text editor like notepad (heaven forbid), which doesn’t have this shortcut. Try not to use the shortcut in your first few HTML projects, so you can build some muscle memory for writing the boilerplate code.

Assignment

  1. Watch and follow along to Kevin Powell’s brilliant Building Your First Web Page video.

  2. Crie alguma memória muscular apagando o conteúdo do index.htmlarquivo e tentando escrever todo o boilerplate novamente de memória. Não se preocupe se você tiver que dar uma olhada no conteúdo da lição nas primeiras vezes se você ficar preso. Apenas continue até que você consiga fazer isso algumas vezes de memória.

  3. Execute seu boilerplate pelo validador HTML do W3 . Os validadores garantem que sua marcação esteja correta e são uma excelente ferramenta de aprendizado, pois fornecem feedback sobre erros de sintaxe que você pode estar cometendo com frequência e dos quais não tem conhecimento, como tags de fechamento ausentes e espaços extras em seu HTML.

Recursos adicionais

Esta seção contém links úteis para conteúdo relacionado. Não é obrigatório, então considere-o suplementar.

  • Outra opção para abrir suas páginas HTML no navegador é usar a extensão Live Preview para VSCode. Isso abrirá seu documento HTML e o atualizará automaticamente toda vez que você salvar o documento. No entanto, recomendamos não usar essa extensão e, em vez disso, fazer isso da maneira antiga, abrindo a página e atualizando-a manualmente no navegador para seus primeiros projetos HTML. Dessa forma, você pode se acostumar com esse processo e não ficará dependente de extensões imediatamente.

  • Se desejar, você pode adicionar o langatributo a elementos individuais em toda a página da web.
