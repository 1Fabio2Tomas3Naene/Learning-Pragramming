Introduo

Todos os documentos HTML tm a mesma estrutura bsica ou boilerplate que precisa estar no lugar antes que qualquer coisa til possa ser feita. Nesta lio, exploraremos as diferentes partes deste boilerplate e veremos como tudo se encaixa.

Criando um arquivo HTML

Para demonstrar um modelo HTML, primeiro precisamos de um arquivo HTML para trabalhar.

Crie uma nova pasta no seu computador e nomeie-a html-boilerplate. Dentro dessa pasta, crie um novo arquivo e nomeie-o index.html.

Voc provavelmente j est familiarizado com muitos tipos diferentes de arquivos, por exemplo, doc, pdf e arquivos de imagem.

Para que o computador saiba que queremos criar um arquivo HTML, precisamos anexar a .htmlextenso ao nome do arquivo, como fizemos ao criar o index.htmlarquivo.

Vale a pena notar que nomeamos nosso arquivo HTML index. Devemos sempre nomear o arquivo HTML que conter a homepage do nosso site index.html. Isso ocorre porque os servidores web, por padro, procuraro por uma index.htmlpgina quando os usurios acessarem nossos sites e no ter uma causar grandes problemas.

O DOCTYPE

Toda pgina HTML comea com uma declarao doctype. O propsito do doctype dizer ao navegador qual verso do HTML ele deve usar para renderizar o documento. A verso mais recente do HTML HTML5, e o doctype para essa verso <!DOCTYPE html>.

Os doctypes para verses mais antigas do HTML eram um pouco mais complicados. Por exemplo, esta a declarao do doctype para HTML4:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

No entanto, provavelmente nunca desejaremos usar uma verso mais antiga do HTML, ento sempre usaremos <!DOCTYPE html>.

Abra o index.htmlarquivo criado anteriormente no seu editor de texto e adicione-o <!DOCTYPE html>na primeira linha.

Elemento HTML

Aps declararmos o doctype, precisamos fornecer um <html>elemento. Isso o que conhecido como elemento raiz do documento, o que significa que todos os outros elementos no documento sero descendentes dele.

Isso se torna mais importante mais tarde, quando aprendemos sobre manipulao de HTML com JavaScript. Por enquanto, saiba apenas que o <html>elemento deve ser includo em todo documento HTML.

De volta ao index.htmlarquivo, vamos adicionar o <html>elemento digitando suas tags de abertura e fechamento, assim:

<!DOCTYPE html>
<html lang="en">
</html>

Notou a palavra langaqui? Ela representa um atributo HTML que associado tag HTML dada, ou seja, <html>neste caso. Esses atributos fornecem informaes adicionais sobre elementos HTML. (Mais sobre isso HTML attributesna prxima lio.)

O que o atributo lang?

langespecifica o idioma do contedo do texto naquele elemento. Este atributo usado principalmente para melhorar a acessibilidade da pgina da web. Ele permite que tecnologias assistivas, por exemplo leitores de tela, se adaptem de acordo com o idioma e invoquem a pronncia correta.

Elemento principal

O <head>elemento onde colocamos meta-informaes importantes sobre nossas pginas da web, e coisas necessrias para que nossas pginas da web sejam renderizadas corretamente no navegador. Dentro do <head>, no devemos usar nenhum elemento que exiba contedo na pgina da web.

De volta ao nosso index.htmlarquivo, vamos adicionar um <head>elemento com um <meta>elemento e um ttulo dentro dele. O <head>elemento vai dentro do <html>elemento e deve ser sempre o primeiro elemento sob a <html>tag de abertura:

<!DOCTYPE html>

<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
  </head>
</html>

Elemento meta

Devemos sempre ter a <meta>tag com a codificao do conjunto de caracteres da pgina da web no <head>elemento: <meta charset="utf-8">.

Definir a codificao muito importante porque garante que a pgina da web exibir smbolos e caracteres especiais de diferentes idiomas corretamente no navegador.

Elemento de ttulo

Outro elemento que sempre devemos incluir no cabealho de um documento HTML o <title>elemento:

<title>My First Webpage</title>

O <title>elemento usado para dar s pginas da web um ttulo legvel por humanos, que exibido na aba do navegador da nossa pgina da web. Por exemplo, se voc olhar para o nome da aba atual do seu navegador, ele dir HTML Boilerplate | The Odin Project; este o <title>do .htmlarquivo atual.

Se no inclussemos um <title>elemento, o ttulo da pgina da web seria o nome do arquivo por padro. No nosso caso, seria index.html, o que no muito significativo para os usurios; isso tornaria muito difcil encontrar nossa pgina da web se o usurio tivesse muitas abas do navegador abertas.

H muitos outros elementos que podem ir dentro do cabealho de um documento HTML. No entanto, por enquanto, crucial saber apenas sobre os dois elementos que cobrimos aqui. Apresentaremos mais elementos que vo para o cabealho ao longo do restante do currculo.

Elemento do corpo

O elemento final necessrio para completar o boilerplate HTML o <body>elemento. aqui que todo o contedo que ser exibido aos usurios ir - o texto, imagens, listas, links e assim por diante.

Para completar o boilerplate, adicione um <body>elemento ao index.htmlarquivo. O <body>elemento tambm vai dentro do <html>elemento e est sempre abaixo do <head>elemento, assim:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
  </head>

  <body>
  </body>
</html>

Visualizando arquivos HTML no navegador

O boilerplate HTML no index.htmlarquivo est completo neste ponto, mas como voc o visualiza no navegador? H algumas opes diferentes:

Usar o Google Chrome

In order to avoid branching our lessons instructions to accommodate for all of the differences between browsers, we are going to be using Google Chrome as our primary browser for the remainder of this course. All references to the browser will pertain specifically to Google Chrome. We strongly suggest that you use Google Chrome for all of your testing going forward.

  1. You can drag and drop an HTML file from your text editor into the address bar of your browser.

  2. You can find the HTML file in your file system and then double click it. This will open up the file in the default browser your system uses.

  3. You can use the terminal to open the file in your browser:

    • Ubuntu: Navigate to the directory containing the file and type google-chrome index.html
    • macOS: Navigate to the directory containing the file and type open ./index.html
    • WSL: Navigate to the directory containing the file and type explorer.exe index.html. Note, this will open up the file in the default browser your system uses.

Using one of the methods above, open up the index.html file we have been working on. Youll notice the screen is blank. This is because we dont have anything in our body to display.

Back in the index.html file, lets add a heading (more on these later) to the body, and save the file:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Webpage</title>
  </head>

  <body>
    <h1>Hello World!</h1>
  </body>
</html>

Now, if you refresh the page in the browser, you should see the changes take effect, and the heading Hello World! will be displayed.

VSCode shortcut

VSCode has a built-in shortcut you can use for generating all the boilerplate in one go. Please note that this shortcut only works while editing a file with the .html extension or a text file with the HTML language already selected. To trigger the shortcut, delete everything in the index.html file and just enter ! on the first line. This will bring up a couple of options. Press the Enter key to choose the first one, and voila, you should have all the boilerplate populated for you.

You may notice that the boilerplate produced by this shortcut includes a line we have not yet mentioned:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

This is not something we need to know about until we discuss responsive design, an advanced topic involving different screen sizes which we will cover much later in the curriculum. For now, you can leave that line as it is.

Its still good to know how to write the boilerplate yourself in case you find yourself using a text editor like notepad (heaven forbid), which doesnt have this shortcut. Try not to use the shortcut in your first few HTML projects, so you can build some muscle memory for writing the boilerplate code.
